# 📱 Riley's App — native iOS wrapper

This folder is a complete native iOS app that wraps the web app in a
full-screen `WKWebView` and adds **on-device, private speech recognition**
(Apple's `SFSpeechRecognizer` with `requiresOnDeviceRecognition = true`).
Riley's voice never leaves the phone.

## ⚠️ Read this first (you have a PC, not a Mac)

**Compiling any native iOS app requires macOS + Xcode.** There is no supported
way to build an iOS app on Windows. So with a **PC + iPhone**, here are your
realistic options:

### ✅ Option 1 — Just use the PWA (recommended, no Mac needed)
The web app already installs on your iPhone and does everything: letters,
sounds, numbers, spelling, tracing, games, rewards, and the parent dashboard.
See the main [README](../README.md) → "Get it on your iPhone." This is the
right choice for one child and costs nothing.

*Only difference vs. native:* the PWA uses Safari's speech recognition instead
of the fully-offline `SFSpeechRecognizer`, and it isn't on the App Store. For
your use case that's a non-issue.

### ☁️ Option 2 — Build in the cloud (no Mac, but advanced)
A GitHub Actions macOS runner compiles this project automatically
(`.github/workflows/ios-build.yml`) so you can confirm the native code builds
without a Mac. **However, installing on a real iPhone still needs code
signing**, which means one of:
- A **free Apple ID** + a Windows tool like **Sideloadly** or **AltStore** to
  sideload the `.ipa` (apps expire every 7 days and must be re-installed), or
- A **paid Apple Developer account ($99/yr)** to distribute via **TestFlight**
  (install on your iPhone from the TestFlight app, no Mac required).

If you want to go this route, tell me and I'll extend the workflow to produce
a TestFlight build (you'd add your Apple credentials as GitHub secrets).

### 🖥️ Option 3 — If you ever get access to a Mac
```bash
brew install xcodegen      # one-time
cd ios
./make-project.sh          # syncs web bundle, generates project, opens Xcode
```
Then press ▶ in Xcode with your iPhone connected.

## What's here
```
project.yml                 XcodeGen spec (generates the .xcodeproj)
make-project.sh             sync web + generate + open (Mac)
scripts/sync-web.sh         copies the web app into RileysApp/Web
RileysApp/
  RileysAppApp.swift        app entry point
  ContentView.swift         full-screen web view
  WebView.swift             WKWebView + native speech message handler
  SpeechBridge.swift        on-device SFSpeechRecognizer bridge
  Info.plist                generated by XcodeGen (mic/speech permissions)
  Assets.xcassets/          app icon
  Web/                      copy of the web app (regenerated by sync-web.sh)
```

## How the speech bridge works
`js/speech.js` checks for `window.webkit.messageHandlers.nativeSpeech`. Inside
this native app that exists, so it posts `"listen"`; `SpeechBridge.swift`
records on-device and calls back `window.__nativeSpeechResult(text)`. In plain
Safari (the PWA) that bridge is absent, so it falls back to the browser's
recognizer automatically. **Same web code, better privacy when native.**
